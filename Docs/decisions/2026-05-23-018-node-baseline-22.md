# ADR-018: Node 런타임 baseline을 22.13+ 로 상향

## 배경

ADR-001은 orchestrator 런타임으로 Node.js + TypeScript를 채택했고, 주변 문서(`Docs/plans/2026-05-22-goal-feature.md`, issue #1 toolchain brief)는 baseline을 "Node 20+"로 기술해 왔다.

ADR-015에서 PipelineEngine 실행 substrate로 Mastra workflow primitives를 채택했다. 그런데 `@mastra/core`는 1.x 전 버전이 `engines.node: ">=22.13.0"`을 선언한다 (1.0.0 ~ 1.36.0 확인). Node 20에서는 Mastra를 설치/실행할 수 없으므로, ADR-015와 "Node 20+" baseline은 양립 불가능하다.

## 옵션

- **Node baseline을 22.13+로 상향**: Mastra 1.x 그대로 채택. Node 22는 현재 LTS(2024-10 진입)라 운영상 리스크 낮음.
- **Node 20을 지원하는 Mastra 버전 선택**: 1.x에 존재하지 않음. ADR-015 재검토를 의미하므로 비현실적.
- **baseline은 두고 engines 미강제**: 미지원 런타임에서 무경고 실행 → 설치/런타임 깨짐 위험만 남김. 거부.

## 결정

Node 런타임 baseline을 **`>=22.13.0`** 으로 상향한다.

- `apps/orchestrator/package.json`의 `engines.node`에 `>=22.13.0` 명시.
- ADR-015 Mastra 채택이 이 상향의 직접 동인이다.

## 이유

- Mastra 1.x 전 버전이 Node 22.13+를 요구 → 선택지 없음.
- Node 22가 활성 LTS라 상향 자체의 리스크가 낮다.
- baseline을 명시적으로 강제해야 미지원 런타임에서의 설치/런타임 실패를 fail-fast로 잡는다.

## 결과

- ADR-001은 유지(런타임 선택은 그대로 Node.js + TypeScript). 본 ADR이 baseline 버전만 갱신한다.
- "Node 20+"를 기술한 문서를 "Node 22.13+"로 갱신한다 (`Docs/plans/2026-05-22-goal-feature.md`).
- Mastra 메이저 업그레이드 시 `engines.node` 재확인.

## 관련

- ADR-001 Node.js + TypeScript 런타임 (런타임 선택 지속, baseline만 갱신)
- ADR-015 Mastra workflow primitives 채택 (상향의 직접 동인)
