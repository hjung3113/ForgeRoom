# Docs/prompts Rules

## 이 폴더의 핵심 규칙

- 에이전트 실행, 리뷰, 구현 조율에 쓰는 프롬프트 문서를 보관한다.
- 이 폴더의 문서는 canonical 설계 문서가 아니라 실행 보조 문서다.
- 프롬프트가 MVP 범위, 모듈 경계, ADR 결정 내용을 바꾸면 원본 문서와 ADR을 먼저 갱신한다.

## 금기

- 원본 설계 문서와 충돌하는 지시를 확정 규칙처럼 쓰지 않는다.
- `Docs/rules/doc-rules.md`에서 금지한 미완성 표식을 남기지 않는다.
- 사용자 승인 없이 ADR 상태를 `decided`로 바꾸라고 지시하지 않는다.

## 체크리스트

- 참조하는 원본 문서 경로가 실제로 존재하는지 확인한다.
- 프롬프트 대상 에이전트의 역할, goal, 금지 범위, 완료 조건을 명확히 쓴다.
- 리뷰 루프와 검증 명령은 관찰 가능한 산출물로 확인 가능하게 쓴다.
