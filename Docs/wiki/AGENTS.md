# Docs/wiki Rules

## 이 폴더의 핵심 규칙

- 리뷰용 HTML wiki와 그 보조 문서를 보관한다.
- 이 폴더의 문서는 기존 설계 문서를 설명하는 파생 산출물이며, canonical source가 아니다.
- MVP 범위, 모듈 경계, ADR 결정 내용을 바꾸려면 원본 문서와 ADR을 먼저 갱신한다.

## 금기

- 원본 설계 문서와 충돌하는 내용을 독자적으로 확정하지 않는다.
- `TBD`, `TODO`, `fill later` 같은 placeholder를 남기지 않는다.
- 리뷰 편의를 위해 만든 요약을 ADR 대체물처럼 취급하지 않는다.

## 체크리스트

- 원본 링크가 실제 문서 경로를 가리키는지 확인한다.
- 비범위와 open question을 명확히 구분한다.
- 리뷰자가 의사결정이 필요한 지점을 찾을 수 있게 표시한다.
